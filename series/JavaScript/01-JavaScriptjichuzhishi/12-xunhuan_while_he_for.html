循环：while 和 for

循环 是一种重复运行同一代码的方法。本节的讨论只限定于基础的循环。

“while”循环

while 循环语法如下：

while (condition) {
  // 代码
  // 所谓的“循环体”
}

当 condition 为真时，执行循环体的code。具体使用案例如下：

let i = 0;
while (i < 3) { // 依次显示 0、1 和 2
  alert( i );
  i++;
}

循环体的单次执行称作 一次迭代。上面的示例中的循环进行了三次迭代。任何表达式或变量都可以是循环条件（condition），而不仅仅是比较。循环条件会被计算，其结果会被转化为布尔值。

ℹ️ 单行循环体不需要大括号
如果循环体只有一条语句，则可以省略大括号 {...}

let i = 3;
while (i) alert(i--);

"do...while"循环

使用 do..while语法可以将条件检查移至循环体下面，语法如下：

do {
  // 循环体
} while (condition);

循环首先执行循环体一次，然后检查条件，当条件为真时重复执行循环体。

使用案例：

let i = 0;
do {
  alert( i );
  i++;
} while (i < 3);

这种循环使用较少，但适用于循环体至少执行一次的场景。

"for" 循环

for 循环是最常用的循环形式。语法如下：

for (begin; condition; step) {
  // ……循环体……
}

下面给出一个示例：

for (let i = 0; i < 3; i++) { // 结果为 0、1、2
  alert(i);
}

上面的代码发生了这些操作：

语句段
beginlet i = 0进入循环时执行一次
conditioni < 3在每次循环迭代之前检查，如果为 false，停止循环
body（循环体）alert(i)条件为真时，重复运行
stepi++在每次循环体迭代后执行

一般循环算法的工作原理如下：

开始运行
→ (如果 condition 成立 → 运行 body 然后运行 step)
→ (如果 condition 成立 → 运行 body 然后运行 step)
→ (如果 condition 成立 → 运行 body 然后运行 step)
→ ...

所以， begin 会在进入循环的时候运行一次，然后不断迭代，检查条件，运行 step

ℹ️ 内联变量声明 这里进行计数的变量 i 是在循环中声明的。这种形式叫做“内联”变量声明。这样声明的变量只在循环中可见。

for (let i = 0; i < 3; i++) {
alert(i); // 0, 1, 2
}
alert(i); // 错误，没有这个变量。

除了在循环中定义变量，我们也可以使用现有的变量：

let i = 0;

for (i = 0; i < 3; i++) { // 使用现有的变量
  alert(i); // 0, 1, 2
}

alert(i); //3，可见，因为是在循环之外声明的

省略语句段

for 循环的任何语句段都可以被省略。

例如，如果我们在循环的开始时不做任何事，那么该部分事实上就可以省略不写：

let i = 0; // 我们已经声明了 i 并对它进行了赋值

for (; i < 3; i++) { // 不再需要 "begin" 语句段
  alert( i ); // 0, 1, 2
}

又或者，你可能需要一个无限循环，那么你甚至可以直接省略所有的内容：

for (;;) {
  // 无限循环
}

但是请注意，两个 ;必须存在，否则会出现语法错误。

跳出循环

通常条件为假时，循环会终止。但有时我们也需要手动去强制循环终止。

JS 提供了 break 指令来强制退出循环。

下面是一个案例：

for(let i = 0; i< 10000; i++){
	if(i > 200){
		break
	}
	console.log(i)
}

在上面这个案例中，当循环迭代到二百的时候，会触发条件语句，执行 break 语句，这条语句会结束掉循环。在一些场景下，这很有用。

继续下一次迭代

continue 指令是 break 的“轻量版”。它不会停掉整个循环。而是停止当前这一次迭代，并强制启动新一轮循环（当条件为真时）。

例子：

for (let i = 0; i < 10; i++) {

  //如果为真，跳过循环体的剩余部分。
  if (i % 2 == 0) continue;

  alert(i); // 1，然后 3，5，7，9
}

上面的案例中，当 i 为偶数的时候会执行 continue 语句，终止当次迭代。所以上述代码只会 alert 奇数的变量 i 。

ℹ️ continue 指令利于减少嵌套
上述案例其实可以不使用 continue 关键字，它同样能实现功能：

for (let i = 0; i < 10; i++) {

  if (i % 2) {
    alert( i );
  }

}

但它多创建了一层嵌套（即 if 代码体）。如果 if 中代码有多行，则可能会降低整体代码的可读性。

⚠️ 禁止在三元运算符 ? 后的表达式中使用 break/continue。
请注意非表达式的语法结构不能和三元运算符一起使用，特别是 break/continue 这样的指令是不允许这样使用的。

(i > 5) ? alert(i) : continue; // continue 不允许在这个位置

上述代码会报错。
在实际开发中，如果你遇到了必须要使用条件去触发 break/continue 时，请使用条件语句来实现。

if (i > 5) {
alert(i);
} else {
continue;
}

这也是不建议使用 ? 运算符代替 if 语句的另一个原因。

❓ JS 中什么是表达式呢？ 在上述的讨论中，我们提到了非表达式不能参与三元运算符，那么什么是表达式呢？在 JavaScript 中，一般说的表达式都是基础表达式，其实包括：

  • 字面量：例如字符串，Number类型，甚至于一个对象，基本类型都属于字面量。
  • 变量：普通带赋值的变量也属于表达式。
  • 函数调用：对于函数的使用，例如：alert("Hello")。
  • 运算表达式：数学表达式，算数、逻辑、位运算等。

break/continue 标签

有时候我们需要一次从多层嵌套的循环中跳出来。在 JS 中，我们可以某一个循环 “命名”，正式的说法称作标签，通过 break/continue 操作符去对标签进行操作，这样描述有些抽象，所以让我们看一个例子：

outer: for (let i = 0; i < 3; i++) {

  for (let j = 0; j < 3; j++) {

    let input = prompt(`Value at coords (${i},${j})`, '');

    // 如果是空字符串或被取消，则中断并跳出这两个循环。
    if (!input) break outer; // (*)

    // 用得到的值做些事……
  }
}

alert('Done!');

在上述例子中，我们为外部循环加上了 outer 标签，后续使用 break 去指向这个标签，这样的方式可以结束外层循环，实现从多层嵌套循环跳出的目的。

也可以将标签移至单独的一行：

outer:
for (let i = 0; i < 3; i++) { ... }

continue 指令也可以与标签一起使用。

⚠️ 标签并不允许“跳到”所有位置
标签不允许我们跳到代码的任意位置。
例如，这样做是不可能的：

break label;  // 跳转至下面的 label 处（无效）

label: for (...)

break 指令必须在代码块内。从技术上讲，任何被标记的代码块都有效，例如：

label: {
// ...
break label; // 有效
// ...
}

……尽管绝大多数情况下 break 都被用于循环内，但是就像上面的案例一样，它也在一些特殊的场景中被使用。
continue 只在循环内部才可用。

总结

本节我们学习了三种循环：

  • while —— 每次迭代之前都要检查条件。
  • do..while —— 每次迭代后都要检查条件。
  • for(;;) —— 每次迭代之前都要检查条件。

通常使用 while(true) 来构造“无限”循环。这样的循环和其他循环一样，也可以通过 break 指令来终止。

如果我们不想再当前循环中做任何事，并且想要转移到下一次循环，那么可以使用 continue 指令。

break/continue 支持循环前的标签。标签是 break/continue 跳出嵌套循环以转到外部的唯一方法。
