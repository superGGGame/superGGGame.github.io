表单

HTML 表单是用来收集用户输入数据的一种方式，它提供了各种不同类型的表单元素，如文本框、复选框、单选按钮、下拉框等。在 web 开发中，表单是非常常见也非常重要的一部分，因为它可以让用户与网站进行交互，提交数据等。

创建表单

使用 <form> 标签来创建一个表单。

表单可以简单的理解为一种获取用户信息的收集，HTML 提供了许多不同形式的用于收集信息的工具。

例如：

<form>
 .
form elements
 .
</form>

在表单中，我们可以添加许多有用的表单元素。

输入框

输入框是表单中最基本的元素之一，用于用户输入文本或数字。它可以通过改变 type 属性而改变形态。

例子：

<form>
  <label for="username">用户名：</label>
  <input type="text" id="username" name="username"><br><br>
</form>

tip: 上面的例子中使用了 <label> 标签，它的作用是描述每个表单元素的用途。

这样你就创建了一个用于输入获取能力的表单。你或许已经注意到了，这里的 <input> 标签 type 属性指定的是 text ，这表示它会渲染为一个输入框。

下面让我们来认识一下 type 属性的具体用法。

输入类型：text

<input type="text"> 定义供文本输入的单行输入字段：

<form>
 First name:<br>
<input type="text" name="firstname">
<br>
 Last name:<br>
<input type="text" name="lastname">
</form>

输入类型：password

<input type="password"> 定义密码字段：

<form>
 User name:<br>
<input type="text" name="username">
<br>
 User password:<br>
<input type="password" name="psw">
</form>

密码字段在输入时会被隐藏，以保护用户信息。

输入类型：submit

<input type="submit"> 定义提交表单数据至表单处理程序的按钮:

<form action="action_page.php">
First name:<br>
<input type="text" name="firstname" value="Mickey">
<br>
Last name:<br>
<input type="text" name="lastname" value="Mouse">
<br><br>
<input type="submit" value="Submit">
</form>

value 属性会定义按钮上的显示值。点击按钮后，表单数据会被提交向 form 表单中，action 属性所指定的网址。

输入类型: radio

<input type="radio"> 定义单选按钮：

<form>
<input type="radio" name="sex" value="male" checked="true">Male
<br>
<input type="radio" name="sex" value="female">Female
</form>

单选按钮除了需要通过 type 指定类型外，还需要为同一组 radio 定义相同的 name 属性。 name 相同的单选按钮会被视为一组，同一组中同一时刻只有一个单选按钮可以被选中。

value 属性是该单选项被选中后的实际文字，相当于选项的实际值，可以和 input 后的值不同。

checked 属性可以指定其被默认选中，该属性值为 true 则被选中，为 false 则不被选中。值得注意的是 checked="true"，即该属性值为 true 时，可以简写为 checked。

输入类型: checkbox

<input type="checkbox"> 定义复选框：

<form>
<input type="checkbox" name="vehicle" value="Bike">I have a bike
<br>
<input type="checkbox" name="vehicle" value="Car">I have a car 
</form>

复选框和单选框类似，同样需要使用 name 属性来指定分组，使用 value 属性来定义选项值。

输入类型: button

<input type="button> 定义按钮：

<input type="button" value="Click Me!">

这也是创建按钮的一种方法。

<textarea> 元素

<textarea> 元素定义多行输入字段（文本域）：

<textarea name="message" rows="10" cols="30">
The cat was playing in the garden.
</textarea>

使用 <textarea> 标签可以创建多行文本框。

<button> 元素

<button> 元素定义可点击的按钮：

<button type="button" onclick="alert('Hello World!')">Click Me!</button>

<select> 元素（下拉列表）

<select> 元素定义下拉列表：

<select name="cars">
<option value="volvo">Volvo</option>
<option value="saab">Saab</option>
<option value="fiat" selected>Fiat</option>
<option value="audi">Audi</option>
</select>

像上面这样来定义下拉框，其中 name 属性是为了给选中值一个名字，而 option 标签创建了下拉列表中的每一个可选项， value 属性定义了可选项的值。

类似 checked ，你也可以使用 selected属性来定义默认的选中值。

表单属性

在前面的内容我们提及了，表单的目的是收集信息。

既然收集了信息，我们必须将信息提交到一个可以收集信息的地方，否则信息存在浏览器中无法发挥价值，表单的作用也就不存在了。

Action属性

action 属性指定表单提交的 URL。当你设置了这个属性后，当你点击了表单中的 type 为 submit 的按钮时，表单会被提交到 action 属性指定的地址。

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="Bill"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Gates"><br><br>
  <input type="submit" value="Submit">
</form>

Target 属性

target 属性规定提交表单后在何处显示响应。

target 属性可设置以下值之一：

值描述
_blank响应显示在新窗口或选项卡中。
_self响应显示在当前窗口中。
_parent响应显示在父框架中。
_top响应显示在窗口的整个 body 中。
framename响应显示在命名的 iframe 中。

默认值为 _self，这意味着响应将在当前窗口中打开。

Method 属性

method 属性指定提交表单数据时要使用的 HTTP 方法。提交表单数据时，默认的 HTTP 方法是 GET。

表单数据可以作为 URL 变量（使用 method="get"）或作为 HTTP post 事务（使用 method="post"）发送。

ℹ️ 关于 GET 的注意事项：

  • 以名称/值对的形式将表单数据追加到 URL
  • 永远不要使用 GET 发送敏感数据！（提交的表单数据在 URL 中可见！）
  • URL 的长度受到限制（2048 个字符）
  • 对于用户希望将结果添加为书签的表单提交很有用
  • GET 适用于非安全数据，例如 Google 中的查询字符串

ℹ️ 关于 POST 的注意事项：

  • 将表单数据附加在 HTTP 请求的正文中（不在 URL 中显示提交的表单数据）
  • POST 没有大小限制，可用于发送大量数据。
  • 带有 POST 的表单提交无法添加书签

总结
